* 欢迎使用我的 Emacs 配置

这是一个现代化、模块化的 Emacs 配置，使用 Org Mode 文学编程编写。
专为 Emacs 29+ 设计，提供丰富的语言支持、LSP 集成和生产力功能。

* 设计理念

** 内置优先
   尽可能使用 Emacs 内置功能，减少不必要的插件

** 模块化设计
   配置按功能拆分为逻辑模块

** 文学编程
   使用 Org Mode 编写，便于文档化和维护

** 性能优化
   延迟加载、快速启动、最小开销

* 🎯 功能特性

** 核心技术栈
   - Emacs 29+ 支持 tree-sitter
   - LSP Bridge 语言服务器集成
   - Vertico + Marginalia + Embark 补全系统
   - Magit Git 操作

** 语言支持
   完整的开发支持：
   - *系统编程*: C/C++, Rust (rust-analyzer), Lua
   - *Web 开发*: TypeScript, JavaScript, HTML/CSS, JSX/React
   - *动态语言*: Python, Bash
   - *标记语言*: Markdown, Org Mode

** 主要功能
   - [[https://github.com/minad/vertico][Vertico]]: 现代化补全
   - [[https://github.com/minad/marginalia][Marginalia]]: 丰富注解
   - [[https://github.com/oantolin/embark][Embark]]: 上下文操作
   - [[https://github.com/manateelazycat/lsp-bridge][LSP Bridge]]: 极速 LSP
   - [[https://magit.vc/][Magit]]: Git 界面
   - [[https://github.com/manateelazycat/blink-search][Blink-search]]: 通用搜索
   - [[https://github.com/manateelazycat/color-rg][Color-rg]]: 快速 grep
   - [[https://github.com/manateelazycat/awesome-tray][Awesome-tray]]: 轻量状态栏
   - [[https://github.com/manateelazycat/emacs-rime][Rime]]: 中文输入
   - [[https://github.com/justbur/emacs-which-key][Which-key]]: 快捷键提示 ⭐
   - [[https://github.com/abo-abo/ace-window][Ace-window]]: 快速窗口切换 ⭐
   - [[https://github.com/abo-abo/avy][Avy]]: 快速跳转导航 ⭐

* 📦 安装

** 前置要求
   - Emacs 29+（推荐）或 Emacs 28+ 配合 tree-sitter
   - Git 用于包管理
   - Node.js 用于 lsp-bridge
   - （可选）Universal Ctags 用于符号导航

** 快速安装

*** 步骤 1: 克隆仓库
    #+begin_src bash
    git clone <repository-url> ~/dotfiles
    cd ~/dotfiles/emacs
    #+end_src

*** 步骤 2: 初始化 Git 子模块
    #+begin_src bash
    make init
    #+end_src

*** 步骤 3: 创建用户配置
    #+begin_src bash
    cp custom-example.el custom.el
    #+end_src

*** 步骤 4: 生成配置
    #+begin_src bash
    make generate
    #+end_src

*** 步骤 5: 链接到 Emacs

**** 方式 A: 符号链接（推荐）
     #+begin_src bash
     # Linux/macOS
     ln -s ~/dotfiles/emacs ~/.emacs.d
     
     # Windows (以管理员身份运行 PowerShell)
     New-Item -ItemType SymbolicLink -Path $env:USERPROFILE\.emacs.d -Target D:\config\dotfiles\emacs
     #+end_src

**** 方式 B: 手动加载
     #+begin_src emacs-lisp
     (load-file "~/dotfiles/emacs/site-lisp/config/init.el")
     #+end_src

*** 步骤 6: 首次启动
    #+begin_src bash
    emacs
    #+end_src

** 故障排除
   #+begin_src bash
   make st        # 检查包状态
   make clean     # 清理生成文件
   make generate  # 重新生成配置
   make h         # 查看所有命令
   #+end_src

* 📦 Git 子模块包管理

本配置使用 Git 子模块管理第三方 Emacs 包。所有包存储在 =site-lisp/extensions/= 目录下，按类别组织。

** 扩展目录组织

扩展按类别子目录组织：

#+begin_src text
site-lisp/extensions/
├── core/           # 核心库 (dash, s.el, f.el, compat, lazy-load)
├── completion/     # 补全系统 (vertico, consult, marginalia, orderless, embark)
├── editor/         # 编辑器功能 (avy, ace-window, vundo, markmacro, fingertip)
├── git/            # Git 集成 (magit, with-editor, jujutsu)
├── input/          # 输入法 (emacs-rime, posframe)
├── lsp/            # LSP 和语言支持 (lsp-bridge, rust-mode, lua-mode 等)
├── org/            # Org-mode 扩展 (org-modern, ox-hugo, org-download)
├── search/         # 搜索工具 (blink-search, color-rg, consult-todo)
├── snippets/       # 代码片段库 (yasnippet, yasnippet-snippets)
├── ui/             # UI 组件 (awesome-tray, themes/, nerd-icons)
├── utils/          # 工具包 (helpful, json-mode, hydra)
└── docs/           # 文档工具 (markdown-ts-mode, flymake-vale)
#+end_src

** 子模块命令

| 命令       | 别名 | 描述                                   |
|------------+------+----------------------------------------|
| make gsb   | init | 初始化所有子模块（引导）               |
| make gsbr  |      | 切换子模块到 main/master 分支          |
| make gsu   | up   | 更新所有子模块到最新                   |
| make gss   |      | 完整同步：初始化 + 切换分支 + 更新     |
| make gsr   |      | 重置子模块到远程 HEAD                  |
| make gsst  | st   | 检查子模块状态                         |
| make gsync |      | 手动编辑后同步 .gitmodules URL         |
| make gsa   |      | 添加新子模块（需要 URL, CAT, NAME）    |
| make gsrm  |      | 移除子模块（需要 CAT, NAME）           |

** 添加新包

将新包添加为 Git 子模块：

#+begin_src bash
# 语法: make gsa URL=<仓库地址> CAT=<类别> NAME=<包名>

# 示例：添加新主题包
make gsa URL=https://github.com/hlissner/emacs-doom-themes CAT=ui/themes NAME=doom-themes

# 示例：添加新编辑器包
make gsa URL=https://github.com/abo-abo/swiper CAT=editor NAME=swiper

# 示例：添加新 LSP 语言支持
make gsa URL=https://github.com/emacs-lsp/lsp-mode CAT=lsp NAME=lsp-mode
#+end_src

添加包后：
1. 子模块被添加到 =.gitmodules=
2. 包被克隆到 =site-lisp/extensions/<CAT>/<NAME>=
3. 提交更改：=git add .gitmodules site-lisp/extensions/<CAT>/<NAME> && git commit=

** 更新包

更新所有包到最新版本：

#+begin_src bash
# 更新所有子模块
make gsu

# 或使用别名
make up
#+end_src

完整同步（推荐在克隆后或子模块不同步时使用）：

#+begin_src bash
# 完整同步：初始化 + 切换到主分支 + 拉取最新
make gss
#+end_src

** 移除包

移除一个包：

#+begin_src bash
# 语法: make gsrm CAT=<类别> NAME=<包名>

# 示例：移除一个包
make gsrm CAT=ui/themes NAME=doom-themes
#+end_src

这将：
1. 取消初始化子模块
2. 从 Git 索引中移除
3. 清理 =.git/modules= 目录

** 检查包状态

#+begin_src bash
# 查看所有子模块状态
make gsst

# 或使用别名
make st
#+end_src

** 最佳实践

1. *选择正确的类别*：将包放在适当的类别目录中
2. *使用描述性名称*：包名应与仓库名匹配
3. *更改后提交*：始终一起提交 =.gitmodules= 和子模块目录
4. *定期更新*：定期运行 =make gsu= 保持包更新
5. *更新后测试*：更新后运行 =make test= 捕获任何破坏性更改

** 模块声明

在 =init.org= 中使用 =mcg!= 宏声明启用的模块：

#+begin_src emacs-lisp
(mcg! :ui
      theme modeline font icons
      
      :editor
      defaults treesit navigation keybindings
      paredit format undo multicursor
      
      :completion
      vertico yasnippet
      
      :tools
      magit search rime tabs eshell
      
      :enhance
      autosave which-key helpful recentf symbol
      
      :org
      org org-agenda org-babel org-capture org-extras
      
      :writing
      markdown vale
      
      :lang
      lsp rust python typescript web cpp lua zig)
#+end_src

* 📝 Org Mode

** 模块化配置系统 ⭐

本配置采用全新的模块化 Org mode 配置系统：

- *统一变量管理* :: 所有变量使用 =mcg-org-*= 命名空间
- *模块化加载* :: 按需加载，智能依赖管理
- *功能开关* :: 通过变量轻松启用/禁用功能
- *错误诊断* :: 内置状态查看和验证工具

** 主要功能

- *Agenda* :: 任务管理和日程安排
- *Capture* :: 快速笔记捕获（多种模板）
- *Export* :: 多格式导出 (HTML, Markdown, PDF)
- *Babel* :: 文档内代码执行（支持 Zig！）✨
- *Hugo* :: 博客发布集成（可选）
- *中文日历* :: 农历、节气、日出日落

** Org Babel 支持的语言

- ✅ Emacs Lisp - Emacs 配置和脚本
- ✅ Python - 数据分析、脚本
- ✅ Shell/Bash - 系统命令
- ✅ Zig - 系统编程 🦎
- ✅ JavaScript - 前端脚本
- ✅ C/C++ - 系统编程
- ✅ SQL - 数据库查询
